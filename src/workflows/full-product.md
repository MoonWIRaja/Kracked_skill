# Workflow: Full Product

> Comprehensive workflow for DEEP scale projects

## Overview
The full-product workflow extends the main workflow with additional rigor. Suitable for complex, high-stakes projects with large teams, sensitive data, or extended timelines.

## Flow

```
Start
  │
  ├── 1. Deep Discovery (/KD-analyze)
  │     └── Comprehensive context gathering
  │     └── Full risk register created
  │     └── Stakeholder mapping
  │     └── Competitive analysis
  │     └── Auto-set scale: DEEP
  │     └── EXIT: Full risk register + stakeholder map
  │
  ├── 2. Detailed Requirements
  │     └── /KD-product-brief → ⏸️ Approval
  │     └── /KD-prd → ⏸️ Approval
  │     └── Detailed user journey mapping
  │     └── Wireframes/mockups consideration
  │     └── EXIT: Comprehensive requirements approved
  │
  ├── 3. Deep Architecture (/KD-architecture --depth=deep)
  │     └── Comprehensive architecture with diagrams
  │     └── Threat modeling
  │     └── Capacity planning
  │     └── All decisions validated + logged
  │     └── ⏸️ Approval
  │     └── EXIT: Architecture approved with full decision log
  │
  ├── 4. Structured Implementation
  │     └── Formal epics and story cards
  │     └── Per-story code reviews
  │     └── Agent Swarm for parallel stories
  │     └── EXIT: All stories complete with per-story reviews
  │
  ├── 5. Comprehensive Quality
  │     └── Full code review (--severity=strict)
  │     └── Full security audit (all OWASP checks)
  │     └── Performance testing
  │     └── Accessibility review
  │     └── EXIT: All quality gates passed
  │
  ├── 6. Staged Deployment
  │     └── Staging deployment + validation
  │     └── Production deployment plan → ⏸️ Approval
  │     └── Production deployment
  │     └── Extended monitoring period
  │     └── EXIT: Production stable
  │
  └── 7. Formal Release
        └── Detailed release notes
        └── Comprehensive scale review
        └── Lessons learned document
        └── Maintenance plan
        └── EXIT: Full cycle closed
```

## Additional Checkpoints for DEEP
1. Product Brief approval
2. PRD approval
3. Architecture approval (with decision log review)
4. Per-story code review (optional)
5. Quality gate review
6. Staging validation
7. Production deployment approval

## When to Use
- Team size 6+
- Timeline > 8 months
- High technical risk
- 6+ integration points
- Sensitive data (PII, financial, health)
- User scale > 10,000
