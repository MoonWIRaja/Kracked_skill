# Checklist: Stage Completion

> Used by all roles to verify stage completion criteria

---

## Stage 1: Discovery
- [ ] Project context fully gathered
- [ ] Scale assessed (SMALL / STANDARD / DEEP)
- [ ] Scale confirmed by user
- [ ] Risks identified and documented
- [ ] `status.md` Meta section populated
- [ ] `status.md` Open Risks populated
- [ ] `status.md` Security Notes populated

## Stage 2: Requirements
- [ ] Product Brief created
- [ ] Product Brief approved ⏸️
- [ ] PRD created
- [ ] PRD approved ⏸️
- [ ] All features have acceptance criteria
- [ ] `status.md` Completed Artifacts updated

## Stage 3: Architecture
- [ ] Architecture document created
- [ ] Tech stack selected with Decision Validation
- [ ] Data models defined
- [ ] API contracts defined
- [ ] Security architecture documented
- [ ] All decisions logged in `status.md`
- [ ] Architecture approved ⏸️

## Stage 4: Implementation
- [ ] Epics and stories created
- [ ] Implementation order defined
- [ ] Dependencies mapped
- [ ] All stories implemented
- [ ] All tests passing
- [ ] Story cards updated

## Stage 5: Quality
- [ ] Code review completed
- [ ] Code review verdict: PASS or PASS WITH CONDITIONS
- [ ] Security audit completed
- [ ] Security verdict: SECURE or CONDITIONALLY SECURE
- [ ] All critical issues resolved

## Stage 6: Deployment
- [ ] Deployment plan created
- [ ] Rollback strategy documented
- [ ] Monitoring configured
- [ ] Health checks defined
- [ ] Production deployment approved ⏸️ (if applicable)
- [ ] Deployment executed
- [ ] Health checks passing

## Stage 7: Release
- [ ] Release notes created
- [ ] Version tagged
- [ ] Post-deployment assessment complete
- [ ] Scale review documented
- [ ] `status.md` fully updated
- [ ] Development cycle closed
